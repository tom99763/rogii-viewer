1. One-Paragraph Summary

This Kaggle competition asks us to predict TVT (true vertical thickness — the geological-depth coordinate) at every measured-depth (MD) point of a horizontal wellbore beyond a known Prediction Start (PS) point. Inputs available at scoring time: the well's 3D trajectory (X, Y, Z, MD), a noisy gamma-ray log (GR, ~29% missing on average), the known TVT before PS, and a paired vertical reference log (typewell) of GR vs TVT. The task is most naturally framed as a geophysical inverse problem: given the observed GR along the lateral and a typewell GR–depth dictionary, recover the wellbore's geological position. Naive baselines reach median per-well RMSE ≈ 10.7 ft; competitive approaches combine signal-alignment (DTW), Tikhonov-regularized GR inversion, sequential filtering, and ML sequence models with auxiliary formation-top supervision.

3. Dataset Snapshot

The dataset is uniform-1-ft-MD-sampled and visually well-organized: each well has exactly two CSVs (horizontal + typewell), the schema is identical across the 773 training wells, and uniform conventions hold for the prediction-start marker (TVT_input goes NaN at PS). The geology of the surveyed area is mostly mild dips — median |dTVT/dMD| is just 0.020 — but a long tail of wells contain steep dips that defeat naive extrapolation.

4. Six Domain Insights (Community + Reference Repo)

Insight 1

This is an inverse problem, not a regression task

Reframe the task as: recover the wellbore position h(MD) = TVT by inverting the forward operator F(h) := GR_v(h(MD)) against the observed horizontal GR. The Tikhonov / Gauss–Newton template solves this in < 200 LOC. See github.com/hhschumann/LWD_inversion for a clean NumPy reference.

Insight 2

Shift & stretch = the geosteering question

A horizontal-well GR snippet matches the typewell GR at the TVT range it was drilled in, modulo a small affine warp for local dip. This is the geosteering engineer's mental algorithm. Algorithmically it is subsequence DTW / cross-correlation on the GR signal.

Insight 3

Nearby wells' typewells are offset copies of the same log

The 773 typewells share only 28 distinct geology labels (dominated by the same 6 formations). Neighbouring wells' typewells are essentially the same vertical GR pattern shifted in depth. Stack the K nearest typewells to get a much cleaner GR reference.

Insight 4

Sequential inversion: march one MD point at a time

The standard LWD workflow reuses the previous MD point's recovered TVT as the initial guess for the next. State-space view: latent TVT evolves with drift = local dip; GR observation is a nonlinear function of the state. Kalman / particle filter variants apply directly.

Insight 5

Formation-top columns leak the answer in training

Columns ANCC, ASTNU, ASTNL, EGFDU, EGFDL, BUDA hold the absolute elevation of each of the six formation tops at the trajectory point. Training-only. Use them as auxiliary multi-task supervision in any ML model — the representations transfer to TVT.

Insight 6

Test set is hidden — code generalizes, not memorizes

The 3 wells under test/ are a submission-shape template. The real evaluation runs on a hidden well set on Kaggle's servers. Implication: never hard-code well IDs; load test/ by directory scan; trust a leave-one-well-out CV over the public LB.

5. Method Portfolio (Verdicts)

MethodVerdictUse it for
Constant Floor (median RMSE ≈ 10.7 ft). First sanity-check submission; metric baseline.
Linear in Z Cuts RMSE roughly in half (median ≈ 35.4 ft). Quick second-pass submission; reuses the lateral's own geometry.
DTW + offset-typewell stacking Strong classical baseline; interpretable; no training. Should ship as your first “real” submission.
Tikhonov Gauss–Newton Mathematically clean; handles missing GR via row-zeroed weights. Needs good initialization (use DTW seed or previous-point solution).
Particle / Kalman filter Robust to multimodal posteriors and missing observations. When inversion exposes local-minimum problems.
ML sequence model Powerful, but 773 wells is “small” without strong priors. Always train with auxiliary heads on the 6 formation-top columns.
Hybrid (inversion → ML residual) Combines physical fidelity with pattern flexibility; the residual is well-behaved. Best ceiling; two-stage but stable.

7. Pipeline Constraints & Pitfalls

  • Hidden test: code must be well-id agnostic — scan the directory.
  • CV: Leave-One-Well-Out or GroupKFold by XY cluster. Random row splits leak.
  • GR missingness is blocky in many wells — use a mask, not mean-imputation.
  • Don't leak the formation-top columns at inference — they are training-only.
  • TVT ≠ TVD. The Z column is surface-relative elevation; TVT is the typewell's vertical coordinate. They differ by a per-region datum offset.
  • Direction is not monotone. A well's TVT can increase, decrease, or stay flat within a single lateral — any model that assumes monotonic TVT will fail on the steep tail.

8. Recommended Roadmap

  1. Reproduce report/baseline_rmse.csv → build a constant-baseline submission.
  2. Build a DTW predictor over the assigned typewell → first credible submission.
  3. Stack 5–10 nearest typewells into a “super-typewell” → re-run DTW.
  4. Implement Tikhonov / Gauss–Newton inversion (see competition_overview.html §5 for the formulation). Initialize from DTW.
  5. Wrap in a sequential filter for online stability.
  6. Train an ML sequence model with formation-top heads as auxiliary supervision.
  7. Ensemble inversion + ML; let ML learn the residual.
