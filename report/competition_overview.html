The ROGII competition looks like a tabular regression task — “predict TVT from features.” It is more usefully framed as a geophysical inverse problem: a horizontal well has drilled through a (partially known) geological structure; we observe its gamma-ray (GR) signature along measured depth (MD); we have a vertical reference GR–depth log (the typewell); we want to recover where along the geological column the well actually is at each MD point — i.e. the per-point TVT. This is the same inverse problem solved (manually or with software like ROGII's StarSteer) every day on a drilling rig.

2. The Physical Setup

A horizontal well is drilled out of a vertical “pilot” or “type” well that penetrates the geological column at one known location. As the bit travels along the lateral, an LWD (Logging While Drilling) gamma-ray tool records GR at every measured-depth (MD) step. Because the rock's GR signature varies vertically (each formation has a characteristic GR pattern), the GR the bit sees tells us which formation it is currently in — and therefore where it is in true vertical thickness (TVT).

Why this is hard. The wellbore is mostly horizontal but the geology is dipping at unknown angles; small dips translate to large TVT changes over a 1–2 km lateral. The GR observed at the bit is noisy. Some segments of GR are missing. And the assigned typewell only covers a finite TVT range.

4. The Geosteering Intuition: Shift & Stretch

Take a short window of horizontal-well GR (red). It has to come from somewhere on the vertical reference (blue). The geological “where am I?” question is answered by finding the depth interval on the typewell that best matches the snippet's GR pattern, allowing for a small affine warp (shift + stretch) to account for slight dip. The matched TVT range is your wellbore's current TVT.

This is exactly what a geosteering engineer does by eye, slide by slide, on the rig. Algorithmically it is subsequence matching with affine alignment — a problem with mature tooling (DTW, sliding correlation, FFT-based phase correlation).

5. Formal Inverse-Problem Formulation

Let m = $h(\mathrm{MD})$ be the unknown TVT sequence we want to recover. Let $d_{obs}$ be the observed horizontal GR vector. Define the forward operator

\[ F(m) := \mathrm{GR}_v\!\bigl(h(\mathrm{MD})\bigr) \]

i.e. “the GR you would expect to see at the bit if it were drilling at the proposed TVT, looked up from the typewell.” If $F(m)$ matched $d_{obs}$ for every MD point, we would be done — but $F$ is approximate (the typewell is noisy and only an analogue of the local geology), so we solve a regularized optimization instead:

\[ \boxed{\min_h \; \Phi(h) \;=\; \underbrace{\|W_d(F(h)-d_{obs})\|_2^2}_{\text{data misfit }\Phi_d} \;+\; \beta \underbrace{\|W_m h\|_2^2}_{\text{smoothness prior }\Phi_m}} \]

$W_d$ is a data-weight matrix (e.g. inverse-variance of GR noise, zero-weight rows for missing GR). $W_m$ encodes a smoothness prior on the TVT path — typically a finite-difference operator so $\|W_m h\|^2$ penalizes curvature. $\beta$ trades fidelity against smoothness and is picked via the L-curve criterion or cross-validation.

Gauss–Newton iteration

Linearize $F$ around the current iterate via its Jacobian $J = \partial F / \partial h$. The gradients of the data-misfit and prior terms are:

\[ \nabla \Phi_d = J^\top W_d^\top W_d (F(h)-d_{obs}), \qquad \nabla \Phi_m = W_m^\top W_m h, \qquad \nabla \Phi = \nabla \Phi_d + \beta \nabla \Phi_m \equiv g. \]

The Gauss–Newton Hessian and the per-iteration model update are:

\[ H_{GN} = J^\top W_d^\top W_d J + \beta W_m^\top W_m, \qquad H_{GN}\, \Delta h = -g. \]

This is exactly the formulation used in the open-source LWD inversion reference repo by Schumann (2021): github.com/hhschumann/LWD_inversion. A faithful 1-D implementation is < 200 lines of NumPy.

Known weakness. Linearized Gauss–Newton is prone to local minima — if the initial guess sits near a different geological layer than the truth, the iteration locks in there. Mitigation: start from the Prediction-Start TVT and march one MD point at a time, using the previous point's solution as the initial guess (next section).

6. Sequential Inversion (Marching Along MD)

The standard LWD-EM workflow marches the inversion forward one MD step at a time, reusing the previous step's recovered TVT as the initialization for the next. This both regularizes the problem (a smoothness prior baked into the iteration order) and dramatically speeds it up (each step needs only a few Gauss–Newton iterations).

In state-space language, this is a sequential Bayesian filter on the latent TVT process:

\[ \underbrace{h_{t} \,=\, h_{t-1} + v_t}_{\text{drift = local dip}} ,\qquad \underbrace{\mathrm{GR}_t \,=\, \mathrm{GR}_v(h_t) + \epsilon_t}_{\text{forward model}} \]

Concrete algorithms that fit this template:

  • Extended Kalman filter — linearize $\mathrm{GR}_v(h)$ around the predicted mean. Cheap and online, but suffers when the GR slope is steep (high $|d\mathrm{GR}_v / dh|$).
  • Unscented Kalman filter — better for the highly nonlinear $\mathrm{GR}_v(h)$ lookup.
  • Particle filter — N hypotheses propagated forward, reweighted by GR likelihood. Robust to multi-modal posteriors (which happen often when GR is locally ambiguous).

7. Two Critical Dataset-Specific Insights

7.1  Nearby wells' typewells are offset copies of the same log

In the survey area, neighbouring wells were drilled into the same dipping geological column. Their assigned typewells therefore record the same underlying GR pattern, just shifted in depth (and mildly stretched by local dip). From the EDA: the 773 training typewells share only 28 distinct geology labels, dominated by the same 6 formations everywhere.

Practical consequence: for any given horizontal well, the assigned typewell is only one of many admissible vertical references. Stacking the nearest K typewells (after a simple cross-correlation alignment) yields a much cleaner reference signal than any single typewell:

Idea. Pre-compute a “super-typewell” for each well by aligning + median-stacking the K nearest typewells (K ≈ 5–10 by XY distance). Use it everywhere the assigned typewell appears in your pipeline. This single change tends to bump inversion-based methods materially.

7.2  The 6 formation-top columns are training-only ground truth — perfect auxiliary signal

In the training horizontal CSVs the columns ANCC, ASTNU, ASTNL, EGFDU, EGFDL, BUDA hold the absolute elevation of each formation top at the current trajectory point. They encode the true geological structure that the model must recover; they are missing in test. Two ways to exploit them:

  1. Auxiliary multi-task targets for any ML model — predict the 6 elevations as extra heads. The internal representations learned have direct geological meaning and transfer to TVT.
  2. Forward-model bootstrap: at training time, you know exactly which formation the bit is in at each MD point. Use that to teach a forward model $F$ (formation, position $\to$ GR) that generalizes beyond the single typewell lookup.

8. Method Portfolio (Graded by Ambition)

MethodWhat it doesProsCons
Baseline
Constant extrapolation		 Predict last known TVT for all post-PS rows. Trivial. Sets the metric floor. Median RMSE ≈ 13 ft (see baseline_rmse.csv).
Baseline
Linear in Z		 Fit TVT ≈ a + b·Z on last 50 known rows, extrapolate. Cuts median RMSE roughly in half versus constant. Fails when the lateral changes dip direction.
Classical
DTW / cross-correlation		 Subsequence-match horizontal GR to typewell GR (or super-typewell). Domain-faithful; no training needed; interpretable. Sensitive to GR noise; needs window-size tuning; offset-typewell stacking helps a lot.
Classical
Tikhonov / Gauss–Newton inversion		 Solve eq. (1)–(6) above with $W_m$ = 2nd-difference smoothness operator. Mathematically clean; supports missing GR via row-zeroed $W_d$. Local-minimum trap; needs good init (use DTW or previous-point solution).
Probabilistic
Particle / Kalman filter		 Sequential state-space inference on TVT with GR likelihood. Handles multimodal posteriors, missing data, online. Particle weights collapse if the forward model is too peaky.
ML
1D-CNN / Bi-LSTM / Transformer		 Sequence-to-sequence regression of TVT from $(\Delta XYZ, \mathrm{GR}, \mathrm{GR}_{mask}, \mathrm{TVT}_{input}, \mathrm{GR}_v(\cdot))$. End-to-end; can absorb soft priors via auxiliary heads (formation tops, geology label, dip). Needs careful per-well CV; 773 wells is “small” for deep models without strong priors.
Hybrid
Inversion seed → ML refinement		 Run classical inversion to produce a TVT trajectory + uncertainty, then learn a residual. Combines physical fidelity with ML's pattern flexibility; the residual is small and well-behaved. Two-stage pipeline; need to manage error propagation.

10. Production Reference: ROGII StarSteer Auto Steer

ROGII's commercial geosteering tool, StarSteer, ships an Auto Steer module that produces exactly the kind of output the competition is asking for: a TVD-vs-MD geological cross-section with the wellbore overlaid, plus a per-MD-point confidence score. This competition is in effect a public benchmark for that auto-interpretation task — winning solutions should be benchmarkable against a real human-quality interpretation produced by the same software.

11. Practical Pitfalls

  • Test set is hidden. The 3 visible test wells under test/ are only a submission template; the real evaluation runs on a hidden set on Kaggle servers. Don't hard-code well IDs; iterate by directory listing.
  • Validate by leave-one-well-out. Random row splits leak (rows within a well are highly auto-correlated). Spatial GroupKFold on XY clusters is the most honest CV.
  • Handle GR missingness with a mask, not mean-imputation. Median GR missing rate is ~28% and the pattern is blocky (long contiguous gaps in some wells).
  • Don't leak the formation-top columns at inference. They are training-only; only use them as auxiliary supervision.
  • TVT is not TVD. It is the depth axis of the typewell, not depth-from-surface. The Z column is surface-relative elevation and differs from TVT by a per-region datum offset.

12. Suggested Roadmap

  1. Reproduce baseline_rmse.csv to know your metric floor. Build a working submission.csv with the constant baseline first.
  2. Build a DTW-based predictor with offset-typewell stacking (insight 7.1). Submit; this should already be competitive on a sizeable fraction of wells.
  3. Implement a 1-D Tikhonov inversion (insight in section 5). Initialize it from DTW.
  4. Wrap that in a sequential filter (section 6) for online stability.
  5. Train a 1-D CNN / Transformer with the 6 formation-top columns as auxiliary heads (insight 7.2).
  6. Hybrid: feed the inversion solution + uncertainty as features to the ML model and let it learn the residual.
